关于go语言Slice中copy用法的一些细节

copy用法

1
2

// 内置函数copy()用于复制一个切片
copy(a, b)  // 后者复制值给前者，长度为二者最小值

复制长度

首先看一个简单的例子：

1
2
3
4
5

a := [...]int{0, 1, 2, 3, 4, 5, 6}
b := a[0:3]
c := make([]int, 2, 2)
copy(c, b)
fmt.Println(c)  \\[0 1]

下面改变c := make()的len和cap：

1
2
3

c := make([]int, 3, 3)
copy(c, b)
fmt.Println(c)  \\[0 1 2]

其中，复制的长度与cap(c)无关，仅与len(c)有关。

复制地址？-> 不！

同样上面一个例子：

1
2
3
4
5
6
7

a := [...]int{0, 1, 2, 3, 4, 5, 6}
b := a[0:3]
c := make([]int, 2, 2)
copy(c, b)
fmt.Println(c)  \\[0 1]
a[0] = 666
fmt.Println(c)  \\[0 1]

显然c用的内存地址与b不同，那么来看一看他们具体的内存地址是什么样的。

1
2

fmt.Printf("b: %p\n", b)   //0xc00000e2c0
fmt.Printf("c: %p\n", c)   //0xc00000a0a0

即a与b共用地址，但是c的内存地址与b不同，也即copy()是值类型。
